  - **features.js Refactor: img+video Layer statt innerHTML-Replace** (`6d44598`): Selbst MIT poster sah Maggy noch "kurzer leerer Screen beim Wechsel". Echter Bug: `screen.innerHTML = '...'` zerstoert das alte `<video>` Element und baut ein neues. Browser zeigt zwischen den DOM-Mutations einen Frame oder mehr **nichts** — selbst mit poster-Attribut, weil das poster erst nach Render des neuen Elements ausgewertet wird. **Lösung**: `screen` enthaelt jetzt zwei persistente Elemente — ein `<img>` (Poster-Layer drunter) und ein `<video>` (drueber, autoplay/loop/muted/playsinline). Beim Klick wird nur `img.src + video.src + video.poster` geaendert, kein Element-Replace mehr. Effekt: das alte img bleibt sichtbar bis das neue Bild geladen ist (oder direkt sichtbar wenn schon im Cache); das Video legt sich oben drauf wenn `play()` greift. Kein leerer DOM-Moment mehr. CSS war schon vorbereitet (`.phone__screen > video, .phone__screen > img { position: absolute; inset: 0; object-fit: cover }` in components.css). Cache-Bust hochgezogen auf `?v=20260502c` fuer features.js + alle mp4/jpg-URLs.
